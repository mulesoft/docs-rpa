= RPA Builder 6.6.0 Release Notes

RPA Builder is the on-premises application that enables you to build automations by specifying the action steps of the process.

include::partial$release-notes/rn-last-version-message.adoc[] 

== April 29, 2024

//new action steps 
* The new Submit Document to MuleSoft IDP action step enables you to send documents to IDP and process them with your published document actions. 
* The new Retrieve Results from MuleSoft IDP action step enables you to retrieve the results of a document processed by IDP after using the Submit Document to MuleSoft IDP action step.
* The new Extract Pages From a PDF File action step enables you to extract, split, and merge PDF files. 
* The new `ErrorOut` global variable enables you to capture error messages from action steps. 
+
You can select between overriding or appending error messages in the new *Error Handling* tab in the workflow initialization settings. 
//enhancements to action steps
* The Read PDF File action step can now read forms and annotations. 
* The Set Variable action step wizard now supports Pin variables.
* You can now specify a custom error message in the Force Error State action step.
//RPA builder updates 
* RPA Recorder is now integrated with RPA Builder, enabling you to start a recording from different sections: 
** From Web Session action steps by using the wizard's *Start Recording* button.
** From the App Session action step by using the wizard's *Start Recording* button.
** From the Workbench by using the *Record* button. 
** From the Workflow by using the *Record Into*, *Record Before*, and *Record After* context menu items. 
* RPA Builder now includes new variable types as Activity Parameters: *Execution ID* and *Connected App Credentials*. 
* RPA Builder now notifies you when a new version is available to download. You can download and install new versions from the *Help* option in the menu bar. 
* The *Help* option in the menu bar now enables you to export log files, enable DEBUG level logs for RPA Builder and RPA Recorder, and select between appending or overwriting trace log files.
//other updates
* The workflow script logfile in analysis packages now includes the current base path.
